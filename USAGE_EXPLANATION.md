## 4. Understanding the Claude Code Usage Report

When you view your active consumption metrics inside Claude Code, your performance is audited across dynamic resource layers. Below is an architectural breakdown of exactly how those metrics are calculated:

### The Token Allocation Core
┌─────────────────────────────────────────────────────────────┐
│ Total Session Input Payload                                 │
├──────────────────────────────┬──────────────────────────────┤
│ 📥 Fresh Prompt Input        │ 💾 Cached Context Reads      │
│ (e.g., 142 Tokens)           │ (e.g., 5.3 Million Tokens)   │
└──────────────────────────────┴──────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 📤 Session Output Generation                                │
├─────────────────────────────────────────────────────────────┤
│ (e.g., 385 Tokens)                                          │
└─────────────────────────────────────────────────────────────┘────────────────────────────────────────────────────────┘


#### 💾 Prompt Cache Reads (`5.3M cache read` vs `124.1k cache write`)
* **What it means:** Claude Code relies heavily on **Prompt Caching**. Instead of compiling your entire repository history, system configurations, and past file changes from scratch on every turn, it stores a snapshot of your project state in server memory. 
* **The Savings Impact:** This report shows an incredibly efficient ratio: **5,300,000 tokens** were read directly from the cache, while only **124,100 tokens** had to be freshly written. Because cached reads cost up to **90% less** than standard input tokens, a multi-million token context operation was completed for a raw cost of just **$1.56**.

#### 📥 Session Inputs (`142 in`)
* **What it means:** This represents the raw, un-cached text payload from your latest message. Keeping this low proves that your environment isn't pushing massive redundant data arrays into the API stream on every execution loop.

#### 📤 Session Outputs (`385 out`)
* **What it means:** This tracks the total token count of the structural code edits, console logs, and diff files generated by Claude during this execution cycle.

## 5. Quota Bucket Mechanics

Claude Code balances your operations across three parallel token registries to keep workflows moving without causing service interruptions:

| Bucket Name | Allocation Focus | Behavioral Mechanics |
| :--- | :--- | :--- |
| **`session-0`** | Short-Term Burst Limit | Governs localized console messaging cycles. This resets continuously over rolling **multi-hour cycles** to let you clear out high-frequency processing spikes. |
| **`weekly_all-1`** | Global Compute Cap | The overarching hard limit tracking overall operational runtime. It completely flushes on a fixed **7-day interval** from your registration date. |
| **`weekly_scoped-2`** | Premium Mode Reserve | Dedicated strictly to tracking higher-tier experimental reasoning modules. This keeps demanding tasks isolated so they don't impact your day-to-day coding tasks. |
